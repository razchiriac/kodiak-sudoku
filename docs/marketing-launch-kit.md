# Marketing launch kit (RAZ-86)

This playbook is intentionally lightweight and repeatable for a solo build.

## Channels to start with

Run only three channels for the first two weeks:

1. X (short clips + challenge links)
2. Reddit (puzzle/sudoku communities with strict self-promo rules)
3. Discord (sudoku + indie dev communities)

## Weekly cadence

- **Monday:** post the weekly challenge angle (daily puzzle + leaderboard CTA).
- **Wednesday:** post one real completion/challenge example.
- **Friday:** post leaderboard recap + invite friends CTA.

## Post templates (10)

Use one hook per template and rotate.

1. **Daily challenge**
   - Hook A: "Today’s Sudoku took me `<time>`. Can you beat it?"
   - Hook B: "Daily puzzle is live. Pure run challenge starts now."
2. **Mistake recovery**
   - Hook A: "Recovered from `<mistakes>` mistakes and still finished in `<time>`."
   - Hook B: "Messy solve, clean finish. Try this run."
3. **No-hint flex**
   - Hook A: "Pure run today: 0 hints, `<time>`."
   - Hook B: "No hints. No excuses. Your turn."
4. **Quick play streak**
   - Hook A: "Short break = `<count>` quick puzzles solved."
   - Hook B: "Built for fast sessions. How many can you clear this week?"
5. **Technique journey**
   - Hook A: "Learning `<technique>` made this puzzle click."
   - Hook B: "Tried one lesson, solved faster immediately."
6. **Friends challenge**
   - Hook A: "Sent a challenge link. Rivalry officially started."
   - Hook B: "Tag a friend who always says Sudoku is easy."
7. **Leaderboard climb**
   - Hook A: "Jumped from `<oldRank>` to `<newRank>` on daily."
   - Hook B: "Climbed the board today. Who’s next?"
8. **Comeback story**
   - Hook A: "Was stuck for 5 minutes, still finished."
   - Hook B: "Almost quit this one. Glad I didn’t."
9. **Build in public**
   - Hook A: "Shipped `<feature>` this week. Feedback wanted."
   - Hook B: "Small feature, big quality-of-life win."
10. **Weekend challenge**
    - Hook A: "Weekend Sudoku duel. Winner picks next puzzle app feature."
    - Hook B: "Drop your time in replies. Fastest run wins bragging rights."

## Community list starter

- r/sudoku (follow posting rules, avoid spammy CTA)
- r/puzzles
- r/webgames (only when updates are substantial)
- Discord: Sudoku communities
- Discord: indie hacker / build-in-public servers

## Outreach DM script (micro-creators)

`Hey <name> — I built a fast web Sudoku app with daily leaderboard + challenge links. I think your audience would enjoy trying one daily run and posting their time. If you want, I can send a direct challenge link + short demo clip.`

## KPI review checklist

Check every Friday:

- Share clicks from completion modal
- Native share success rate
- Clipboard fallback success rate
- Sessions with `utm_source=sudoku_app&utm_medium=share`
- Shared-link visitors who start a puzzle
- Shared-link visitors who complete a puzzle

Keep only channels that deliver quality sessions and completions.
